**Part 1 (Question):**

Consider the following Python code:

```python
import asyncio

async def task(name):
    print(f"Task {name} started")
    await asyncio.sleep(2)
    print(f"Task {name} completed")

async def main():
    tasks = [task(f'Task-{i}') for i in range(5)]
    await asyncio.gather(*tasks)

if __name__ == "__main__":
    import time
    start_time = time.time()
    asyncio.run(main())
    end_time = time.time()
    print(f"Total execution time: {end_time - start_time} seconds")
```

What will be the output of this code and what does it demonstrate about asyncio?

A) The tasks will run sequentially, taking 10 seconds in total.

B) The tasks will run concurrently, and the total execution time should be close to 2 seconds.

C) There will be an error because asyncio.sleep() is not a coroutine function.

D) The tasks will run concurrently, but the total execution time will exceed 2 seconds due to context switching overhead.

[ANSWER_SEPARATOR] 
**Part 2 (Answer):**

B) The tasks will run concurrently, and the total execution time should be close to 2 seconds.

Explanation: This question tests the understanding of asyncio's ability to run coroutines concurrently. When you use `asyncio.gather`, it schedules all tasks at once and waits for them to complete, making the overall execution time approximately equal to the longest-running task (in this case, each task takes 2 seconds). The key point demonstrated here is that asyncio allows for efficient concurrent execution of I/O-bound tasks, which can lead to significant performance improvements in applications with multiple such tasks.
