Consider the following Python code snippet that uses decorators and metaclasses together:

```python
class Meta(type):
    def __new__(cls, name, bases, dct):
        print(f"Creating class {name}")
        return super().__new__(cls, name, bases, dct)

def decorator(func):
    def wrapper(*args, **kwargs):
        print("Decorator is running")
        return func(*args, **kwargs)
    return wrapper

@decorator
class MyClass(metaclass=Meta):
    def __init__(self):
        self.value = 42

if __name__ == "__main__":
    obj = MyClass()
```

What happens when you run this code? Explain the sequence of events and outputs.

A) "Decorator is running" will be printed, then "Creating class MyClass"

B) "Creating class MyClass" will be printed, then "Decorator is running"

C) An error will occur because decorators cannot be applied to classes

D) "Decorator is running" will not be printed, only "Creating class MyClass"

**Part 2 (Answer):**

A) "Decorator is running" will be printed, then "Creating class MyClass"

Explanation: The `Meta` metaclass is used to define how the class should be created. When `MyClass` is defined, Python first calls the metaclass's `__new__` method to create the class object. After the class is created, Python then applies decorators to the methods of the class. Therefore, "Decorator is running" is printed before "Creating class MyClass".
