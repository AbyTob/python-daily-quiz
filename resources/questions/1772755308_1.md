Consider the following Python code snippet:

```python
import asyncio

class AsyncLogger:
    def __init__(self, filename):
        self.filename = filename

    async def log(self, message):
        with open(self.filename, 'a') as file:
            await file.write(message + '\n')

async def main():
    logger = AsyncLogger('log.txt')
    await asyncio.gather(
        logger.log('Message 1'),
        logger.log('Message 2'),
        logger.log('Message 3')
    )

# Run the main coroutine
asyncio.run(main())
```

Which of the following statements is true about this code?

A) The `AsyncLogger` class does not support asynchronous file writing.
B) The `log` method will execute concurrently, potentially appending messages in any order.
C) The `with` statement inside the `log` method ensures that each message is logged on a new line.
D) All messages are guaranteed to be appended to 'log.txt' in the order they were called.
    
    ## Answer
    Part 2 (Answer):
B) The `log` method will execute concurrently, potentially appending messages in any order.

Explanation:
- The `AsyncLogger` class is defined with an asynchronous `log` method that uses a `with` statement for file writing. This ensures that the file operations are thread-safe and handle context correctly.
- When `asyncio.gather` is used to call multiple instances of `logger.log`, it executes these calls concurrently, meaning that the order in which messages are appended to the file is not guaranteed. This demonstrates an important aspect of asynchronous programming where tasks can be interleaved.

This question tests understanding of asynchronous IO and the implications of concurrent execution, rather than simply whether `asyncio` or file operations are used correctly.
