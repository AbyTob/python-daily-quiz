Consider the following Python code snippet that attempts to create a decorator to measure execution time of a function:

```python
import time

def timing_decorator(func):
    def wrapper(*args, **kwargs):
        start_time = time.time()
        result = func(*args, **kwargs)
        end_time = time.time()
        print(f"{func.__name__} executed in {end_time - start_time:.4f}s")
        return result
    return wrapper

@timing_decorator
def compute_sum(n):
    return sum(range(n))
```

Which of the following statements is true regarding this code snippet?

A) The `compute_sum` function will execute in a significantly longer time due to the additional function call made by the decorator.

B) The `timing_decorator` will measure the execution time accurately, as it correctly captures both the start and end times.

C) The `wrapper` function inside the decorator will not capture all arguments passed to `compute_sum`.

D) The use of `func.__name__` ensures that the original function name is printed even after decoration.
    
    ## Answer
    B) The `timing_decorator` will measure the execution time accurately, as it correctly captures both the start and end times.

Explanation: The decorator correctly calculates the execution time by measuring the time before and after calling the function. The `wrapper` function passes all arguments to the original function using `*args` and `**kwargs`, ensuring that the original function behaves as expected. The use of `func.__name__` ensures that the name of the decorated function is preserved, which is a common requirement in decorators.
