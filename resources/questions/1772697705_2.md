    ## Question
    [QUESTION] Consider the following Python code that attempts to create a singleton pattern using a metaclass:

```python
class SingletonMeta(type):
    _instances = {}

    def __call__(cls, *args, **kwargs):
        if cls not in cls._instances:
            instance = super(SingletonMeta, cls).__call__(*args, **kwargs)
            cls._instances[cls] = instance
        return cls._instances[cls]

class MyClass(metaclass=SingletonMeta):
    def __init__(self, value):
        self.value = value

# Usage
obj1 = MyClass(10)
obj2 = MyClass(20)

print(obj1.value)  # Output?
print(obj2.value)  # Output?
```

What will be the output of the above code when executed?

A) 10, 20  
B) 10, 10  
C) Error: Singleton pattern not implemented correctly  
D) Error: Cannot instantiate a singleton class directly
    
    ## Answer
    [B] 10, 10

Explanation:
The `SingletonMeta` metaclass ensures that only one instance of the `MyClass` is created. When an instance of `MyClass` is requested for the first time (`obj1 = MyClass(10)`), it creates a new instance and stores it in `_instances`. When `obj2 = MyClass(20)` is called, it checks if `MyClass` already exists in `_instances`. Since it does, it returns the existing instance (`obj1`) without creating a new one. Therefore, both `obj1.value` and `obj2.value` will be 10, demonstrating that only one instance of `MyClass` was created.
