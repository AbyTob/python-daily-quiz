Consider the following Python code:

```python
import asyncio

async def fetch_data():
    await asyncio.sleep(1)
    return "Data fetched"

class AsyncDecorator:
    def __call__(self, func):
        async def wrapper(*args, **kwargs):
            result = await func(*args, **kwargs)
            return f"Processed {result}"
        return wrapper

@AsyncDecorator()
async def main():
    data = await fetch_data()
    return data

# Running the code
loop = asyncio.get_event_loop()
result = loop.run_until_complete(main())
print(result)
```

What will be printed to the console when the above code is executed?

A) Data fetched  
B) Processed Data fetched  
C) Data processed  
D) Error: not awaited
    
    ## Answer
    B) Processed Data fetched  

Explanation:
In this code, we have an `AsyncDecorator` class that wraps any asynchronous function it decorates. The `wrapper` method inside the decorator takes care of calling the original function and then processes its result by prefixing it with "Processed ". 

When `main` is decorated with `@AsyncDecorator()`, the call to `main()` triggers the `fetch_data` coroutine, which simulates a delay using `asyncio.sleep(1)`. Once the data is fetched, the `wrapper` method of the decorator processes this result and returns "Processed Data fetched". 

Therefore, when you run the code, it will print "Processed Data fetched" to the console.
