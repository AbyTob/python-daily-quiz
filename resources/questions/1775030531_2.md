Consider the following Python code snippet:

```python
import asyncio

async def fetch_data(url):
    print(f"Fetching data from {url}")
    await asyncio.sleep(2)
    return f"Data from {url}"

async def main():
    url = "http://example.com"
    task = asyncio.create_task(fetch_data(url))
    
    while not task.done():
        await asyncio.sleep(1)
        print("Waiting for data...")
    
    result = await task
    print(result)

# Run the event loop
asyncio.run(main())
```

What is the output of this code? Choose one option:

A) Fetching data from http://example.com  
   Waiting for data...  
   Data from http://example.com

B) Fetching data from http://example.com  
   Data from http://example.com  

C) Waiting for data...  
   Fetching data from http://example.com  
   Data from http://example.com  

D) Waiting for data...  
   Fetching data from http://example.com  
   Waiting for data...  
   Data from http://example.com
    
    ## Answer
    **Part 2 (Answer):**

C) Waiting for data...  
   Fetching data from http://example.com  
   Data from http://example.com  

Explanation:

- When `asyncio.create_task(fetch_data(url))` is called, it schedules the coroutine to run in the background and returns a Task object.
- The while loop checks if the task is done using `task.done()`. Since the task has just been created but not yet started, it returns `False`, so the loop prints "Waiting for data..." once before awaiting the task completion.
- The `await asyncio.sleep(1)` inside the loop makes the event loop yield control back to the main thread, allowing other tasks to run if any. In this case, there are no other tasks, but in a real scenario, it could allow other coroutines or callbacks to execute during this sleep period.
- After 2 seconds (simulated by `await asyncio.sleep(2)` inside `fetch_data`), the task completes and the result is printed.
