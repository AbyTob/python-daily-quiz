Consider the following Python code snippet that utilizes a metaclass to create a class with dynamically generated attributes based on its base classes:

```python
class Meta(type):
    def __new__(cls, name, bases, dct):
        for base in bases:
            if hasattr(base, '_attributes'):
                for attr_name, value in base._attributes.items():
                    dct[attr_name] = value
        return super().__new__(cls, name, bases, dct)

class BaseA(metaclass=Meta):
    _attributes = {'a': 10}

class BaseB(metaclass=Meta):
    _attributes = {'b': 20}

class Derived(BaseA, BaseB):
    pass
```

What will be the output of `Derived.a + Derived.b`?

A) 30  
B) Error: 'Derived' has no attribute 'a' or 'b'  
C) AttributeError: type object 'Derived' has no attribute '__module__'  
D) TypeError: unsupported operand type(s) for +: 'type' and 'int'
    
    ## Answer
    [A] 30

Explanation:
The metaclass `Meta` dynamically adds attributes from its base classes to the class being created. When `Derived` is instantiated, it inherits from both `BaseA` and `BaseB`, which have `_attributes` dictionaries. The metaclass merges these dictionaries into `Derived`'s dictionary, resulting in `Derived.a = 10` and `Derived.b = 20`. Therefore, `Derived.a + Derived.b` evaluates to `30`.
