You are tasked with creating a Python application that manages a simple task queue. The queue should be able to handle tasks asynchronously and ensure that only one task is running at a time, regardless of the number of concurrent requests.

To achieve this, you decide to use a metaclass to control the behavior of the task queue class. You also want to use asyncio for handling asynchronous operations.

Here's your initial code:

```python
import asyncio

class TaskQueueMeta(type):
    def __init__(cls, name, bases, dct):
        super().__init__(name, bases, dct)
        cls._queue = []
        cls._running_task = None

    async def run_tasks(cls):
        while True:
            if not cls._queue and cls._running_task is None:
                await asyncio.sleep(1)  # Wait for a task to arrive
            elif cls._queue and cls._running_task is None:
                cls._running_task = cls._queue.pop()
                await cls._running_task()
                cls._running_task = None

class TaskQueue(metaclass=TaskQueueMeta):
    @classmethod
    async def add_task(cls, task_func):
        cls._queue.append(task_func)
        if not cls._running_task:
            asyncio.create_task(cls.run_tasks())

# Example usage
async def my_task():
    print("Running task")

async def main():
    await TaskQueue.add_task(my_task)
    await asyncio.sleep(2)

asyncio.run(main())
```

What issues do you expect with the current implementation, and how would you improve it to ensure that only one task is running at a time?

A) The `run_tasks` method will not correctly handle exceptions if a task raises an error.

B) Using `create_task` inside the loop might lead to creating too many tasks, leading to resource exhaustion.

C) The `TaskQueueMeta` metaclass does not provide a way to cancel ongoing tasks gracefully.

D) The `add_task` method should be synchronized to ensure that only one task is added at a time when multiple threads call it concurrently.

C) The `TaskQueueMeta` metaclass does not provide a way to cancel ongoing tasks gracefully.

Explanation:
The current implementation of the `TaskQueueMeta` metaclass and its associated methods lacks a mechanism for canceling ongoing tasks. If a task raises an exception or if the application needs to be shut down, there is no straightforward way to stop the running task gracefully.

To improve this, you can add methods to the `TaskQueue` class that allow cancelling tasks:

```python
class TaskQueue(metaclass=TaskQueueMeta):
    @classmethod
    async def cancel_task(cls):
        if cls._running_task:
            cls._running_task.cancel()
            try:
                await cls._running_task
            except asyncio.CancelledError:
                pass

    @classmethod
    async def add_task(cls, task_func):
        cls._queue.append(task_func)
        if not cls._running_task:
            asyncio.create_task(cls.run_tasks())
```
