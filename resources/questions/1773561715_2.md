Consider the following Python code snippet that uses a metaclass and decorator to modify class behavior dynamically:

```python
from functools import wraps

def add_method(cls, name, func):
    setattr(cls, name, func)
    return cls

class Meta(type):
    def __new__(cls, name, bases, dct):
        dct = {key: value for key, value in dct.items() if not callable(value)}
        return super().__new__(cls, name, bases, dct)

@add_method(Meta, 'greet', lambda self: f"Hello from {self.name}")
class MyClass(metaclass=Meta):
    def __init__(self, name):
        self.name = name

if __name__ == "__main__":
    obj = MyClass("Alice")
    print(obj.greet())
```

Which of the following statements about this code is true?

A) The `add_method` decorator dynamically adds a method to a class at runtime.
B) The `Meta` metaclass filters out all callable attributes from the class dictionary during class creation.
C) When an instance of `MyClass` is created, it automatically calls the `greet` method due to metaclass magic.
D) This code will cause a `TypeError` because `add_method` cannot modify the metaclass in place.
    
    ## Answer
    Part 2 (Answer):
A) The `add_method` decorator dynamically adds a method to a class at runtime.

Explanation: In this code, the `add_method` function is used as a decorator to add a new method `greet` to the class `MyClass`. The metaclass `Meta` ensures that only non-callable attributes are added to the class dictionary, preventing callable methods like `greet` from being included. When an instance of `MyClass` is created and its `greet` method is called, it behaves as expected, demonstrating dynamic class modification.
