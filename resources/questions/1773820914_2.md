Consider the following Python code snippet that demonstrates a decorator designed to measure execution time of a function:

```python
import time

def timing_decorator(func):
    def wrapper(*args, **kwargs):
        start_time = time.time()
        result = func(*args, **kwargs)
        end_time = time.time()
        print(f"Function {func.__name__} took {end_time - start_time:.4f} seconds to execute.")
        return result
    return wrapper

@timing_decorator
def compute_sum(n):
    return sum(range(n))

# Example usage:
result = compute_sum(1000000)
```

Which of the following statements is true regarding this code?

A) The decorator `timing_decorator` will only measure execution time if a function explicitly calls it within its body.

B) When applied to `compute_sum`, the decorator adds functionality to print the execution time, but does not change the return value of the original function.

C) The decorator `timing_decorator` requires the use of metaclasses to be applied to class methods.

D) Applying this decorator to any function will cause an infinite recursion because it calls itself without a base case.
    
    ## Answer
    B) When applied to `compute_sum`, the decorator adds functionality to print the execution time, but does not change the return value of the original function.

Explanation: The `timing_decorator` is correctly implemented as a function decorator that wraps another function and measures its execution time. It prints the time taken for execution and returns the result of the wrapped function unchanged. This demonstrates how decorators can be used to add functionality to existing functions without modifying their internal implementation.
