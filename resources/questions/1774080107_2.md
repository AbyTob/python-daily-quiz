Consider the following Python code:

```python
import asyncio

class AsyncDecorator:
    def __init__(self, func):
        self.func = func

    def __call__(self, *args, **kwargs):
        return asyncio.run(self.func(*args, **kwargs))

@AsyncDecorator
async def async_task(delay):
    await asyncio.sleep(delay)
    print(f"Task completed after {delay} seconds")

async def main():
    task1 = async_task(2)
    task2 = async_task(3)
    await asyncio.gather(task1, task2)

if __name__ == "__main__":
    asyncio.run(main())
```

What is the output of running this script?

A) Task completed after 2 seconds
   Task completed after 3 seconds

B) Task completed after 5 seconds

C) The program will raise an exception

D) None of the above
    
    ## Answer
    A) Task completed after 2 seconds  
Task completed after 3 seconds

Explanation: The `AsyncDecorator` class is used to make a synchronous function into an asynchronous one by running it within an asyncio event loop. When `async_task` is called with arguments, the decorator uses `asyncio.run()` to execute the decorated coroutine and wait for it to complete. Since `asyncio.gather()` is used in the `main()` function to run both tasks concurrently, each task will start immediately after the previous one finishes, leading to a total completion time of 5 seconds (2 + 3). Therefore, the output will be "Task completed after 2 seconds" followed by "Task completed after 3 seconds".
