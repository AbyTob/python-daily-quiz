Consider the following Python code snippet that uses a metaclass to add a method dynamically to all classes defined within it:

```python
class DynamicMethodAdder(type):
    def __new__(cls, name, bases, dct):
        dct['dynamic_method'] = lambda self: f"Added dynamically to {name}"
        return super().__new__(cls, name, bases, dct)

class BaseClass(metaclass=DynamicMethodAdder):
    pass

class SubClass(BaseClass):
    pass
```

Which of the following statements is true about the behavior of `SubClass` after its definition?

A) `SubClass.dynamic_method()` will raise an AttributeError because dynamic_method was not defined at class creation.

B) `SubClass.dynamic_method()` will return "Added dynamically to SubClass" when called on an instance of `SubClass`.

C) `BaseClass.dynamic_method()` can be accessed directly from the `BaseClass` without needing an instance.

D) Calling `dynamic_method` on any object that is not a subclass of `DynamicMethodAdder` will result in a TypeError.
    
    ## Answer
    [B] SubClass.dynamic_method() will return "Added dynamically to SubClass" when called on an instance of SubClass.
