Consider the following Python code snippet:

```python
import asyncio

class AsyncDecorator:
    def __init__(self, func):
        self.func = func

    def __call__(self, *args, **kwargs):
        return asyncio.run(self.func(*args, **kwargs))

@AsyncDecorator
async def my_async_function(x):
    await asyncio.sleep(1)
    return x * 2

def run():
    result = my_async_function(5)
    print(result)

if __name__ == "__main__":
    run()
```

What will be the output of this script when executed?

A) `10`
B) An error
C) The program will hang forever
D) None
    
    ## Answer
    A) `10`

Explanation:
The `AsyncDecorator` class is designed to make any function asynchronous by running it within an asyncio event loop. In the provided code, `my_async_function` is decorated with `@AsyncDecorator`, which means when `my_async_function(5)` is called, it will be executed asynchronously. Inside `run()`, calling `my_async_function(5)` triggers the decorator, which then runs `my_async_function` within an asyncio event loop using `asyncio.run()`. This allows `asyncio.sleep(1)` to execute correctly, sleeping for 1 second before returning `x * 2`, which is `10`. Therefore, the output will be `10`.
