Consider the following code snippet:

```python
import threading

def thread_decorator(func):
    def wrapper(*args, **kwargs):
        thread = threading.Thread(target=func, args=args, kwargs=kwargs)
        thread.start()
        return thread
    return wrapper

@thread_decorator
def print_numbers(n):
    for i in range(n):
        print(i)

def main():
    threads = [print_numbers(5) for _ in range(3)]
    for thread in threads:
        thread.join()

if __name__ == "__main__":
    main()
```

What is the issue with this code, and how can it be fixed to ensure that all numbers from 0 to 14 are printed without any duplication?

A) The `thread_decorator` does not handle the order of execution correctly. It should use a thread pool instead.

B) The `print_numbers` function is not thread-safe. Multiple threads might write to standard output simultaneously, causing duplication.

C) The `join` method is called on the threads too early, preventing any work from being done.

D) The `thread_decorator` does not return the correct type from the wrapper function.
    
    ## Answer
    B) The `print_numbers` function is not thread-safe. Multiple threads might write to standard output simultaneously, causing duplication.

The issue with the code lies in the fact that Python's print function is not thread-safe. When multiple threads attempt to print to the same output stream (in this case, the console), it can lead to interleaved outputs and duplicated numbers.

To fix this, you can use a lock from the `threading` module to ensure that only one thread prints at a time. Here's how you could modify the code:

```python
import threading

lock = threading.Lock()

def print_numbers(n):
    for i in range(n):
        with lock:
            print(i)

@thread_decorator
def print_numbers_decorated(n):
    for i in range(n):
        with lock:
            print(i)

def main():
    threads = [print_numbers(5) for _ in range(3)]
    for thread in threads:
        thread.join()

if __name__ == "__main__":
    main()
```

By using the `lock` object, we ensure that only one thread can execute the `with lock:` block at a time, preventing any duplication of print statements.
