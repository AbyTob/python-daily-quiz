Consider the following Python code:

```python
import asyncio

async def fetch_data(url):
    print(f"Fetching data from {url}")
    await asyncio.sleep(2)
    return f"Data from {url}"

class AsyncTaskManager:
    tasks = []

    @classmethod
    async def run_tasks(cls, *urls):
        for url in urls:
            task = asyncio.create_task(fetch_data(url))
            cls.tasks.append(task)

        results = await asyncio.gather(*cls.tasks)
        return results

async def main():
    manager = AsyncTaskManager()
    await manager.run_tasks("http://example.com", "http://example.org")
    print(manager.tasks[0].result())
    print(manager.tasks[1].result())

if __name__ == "__main__":
    asyncio.run(main())
```

What is the output of this code? Explain why.

A) It will raise an error because `manager.tasks[0].result()` and `manager.tasks[1].result()` are called before the tasks are completed.  
B) It will print "Fetching data from http://example.com", "Fetching data from http://example.org", then "Data from http://example.com" and "Data from http://example.org".  
C) It will print "Fetching data from http://example.com", then "Fetching data from http://example.org", but it will not wait for the tasks to complete before printing.  
D) It will raise an error because `manager.tasks` is a class variable, and it should be accessed using the class name rather than the instance.

**Part 2 (Answer):**

B) It will print "Fetching data from http://example.com", "Fetching data from http://example.org", then "Data from http://example.com" and "Data from http://example.org".

Explanation:
The code uses an asynchronous context manager to manage tasks. When `run_tasks` is called, it creates tasks using `asyncio.create_task`, which schedules the task for execution but does not wait for it to complete. The tasks are then gathered using `asyncio.gather`, which waits for all tasks to complete before proceeding.

In the `main` function, after running the tasks, calling `manager.tasks[0].result()` and `manager.tasks[1].result()` retrieves the results of the completed tasks, which are then printed. Since the tasks have already completed by the time their results are accessed, they do not raise an error when their results are retrieved.

Therefore, the output will be "Fetching data from http://example.com", "Fetching data from http://example.org", "Data from http://example.com", and "Data from http://example.org".
