    **Part 1 (Question):**

Consider the following Python code snippet:

```python
import asyncio

async def fetch_data(task_id):
    print(f"Fetching data for task {task_id}")
    await asyncio.sleep(2)
    return f"Data for task {task_id}"

async def main():
    tasks = [fetch_data(i) for i in range(5)]
    results = await asyncio.gather(*tasks)
    print(results)

# Run the main function to see the output
asyncio.run(main())
```

What is the expected output of this code when executed, and why?

A) It will raise an error because `fetch_data` is not a coroutine  
B) It will fetch data for tasks 0 through 4 concurrently and print the results in order  
C) It will fetch data for tasks 0 through 4 sequentially (one after another)  
D) The program will hang indefinitely without completing

**Part 2 (Answer):**

The correct answer is B) It will fetch data for tasks 0 through 4 concurrently and print the results in order.

Explanation:  
In the provided code, `async def main():` creates a list of coroutines using a list comprehension. When you pass these coroutines to `await asyncio.gather(*tasks)`, it schedules all of them to run concurrently. The `gather` function then waits for all tasks to complete and returns their results as a list. Since the tasks are scheduled concurrently, the program does not wait for each task to finish before starting the next one, thus achieving concurrent execution. Once all tasks are completed, the results are printed in the order they were started (from `task_0` to `task_4`).
