Consider the following Python code that uses a metaclass to automatically add a `__str__` method to all classes it decorates. The goal is to provide a simple string representation of each class instance, showing its class name and its attributes.

```python
class StringReprMeta(type):
    def __new__(cls, name, bases, dct):
        if '__str__' not in dct:
            dct['__str__'] = lambda self: f"{self.__class__.__name__}({', '.join(f'{k}={v}' for k, v in self.__dict__.items())})"
        return super().__new__(cls, name, bases, dct)

class Person(metaclass=StringReprMeta):
    def __init__(self, name, age):
        self.name = name
        self.age = age

# Example usage:
person = Person("Alice", 30)
print(person)  # Expected output: Person(name=Alice, age=30)
```

Which of the following statements is true about this code?

A. The `StringReprMeta` metaclass will not add a `__str__` method if it already exists in the class being decorated.

B. If a class uses `Person` as its base class, it will automatically have a `__str__` method added by `StringReprMeta`.

C. The `StringReprMeta` metaclass correctly implements a descriptor for each attribute to provide string representation.

D. When creating an instance of `Person`, the `__init__` method is called before the `__str__` method, so the attributes are set first.
    
    ## Answer
    A. The `StringReprMeta` metaclass will not add a `__str__` method if it already exists in the class being decorated.

Explanation:
The correct answer is A. In the provided code, the `StringReprMeta` metaclass checks if the `__str__` method is present in the dictionary of the class being created (`dct`). If it is not found, it adds a lambda function that constructs a string representation based on the class name and its attributes. If the `__str__` method already exists, the metaclass will not overwrite it. This behavior ensures that the `__str__` method is only added if necessary, preventing any potential issues with overriding an existing method.
