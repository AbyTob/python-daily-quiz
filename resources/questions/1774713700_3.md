Consider the following Python code:

```python
import asyncio

class AsyncDecorator:
    def __init__(self, func):
        self.func = func

    def __call__(self, *args, **kwargs):
        return asyncio.run(self.func(*args, **kwargs))

@AsyncDecorator
async def fetch_data():
    await asyncio.sleep(1)
    return "Data fetched"

async def main():
    result = await fetch_data()
    print(result)

# Run the main function to see the output
# Uncomment the following line to execute
# asyncio.run(main())
```

What will be printed when the code is executed? (Assume the code is run in an environment where asyncio is available.)

A) Data fetched  
B) Nothing  
C) Runtime error  
D) A syntax error
    
    ## Answer
    C) Runtime error  

Explanation:
The `AsyncDecorator` attempts to execute an asynchronous function synchronously using `asyncio.run()`. When `fetch_data()` is called within the decorator, it's not awaited properly, leading to a `RuntimeError` because `asyncio.run()` cannot be called from within another asyncio event loop. The correct way to use this decorator would involve ensuring that `fetch_data()` is run within an asyncio event loop, or by modifying the decorator to handle async functions correctly.
