Consider the following Python code:

```python
import asyncio

class AsyncManager:
    def __init__(self, tasks):
        self.tasks = tasks

    async def execute(self):
        results = []
        for task in self.tasks:
            result = await task()
            results.append(result)
        return results

async def task1():
    await asyncio.sleep(0.5)
    return 1

async def task2():
    await asyncio.sleep(1)
    return 2

async def main():
    manager = AsyncManager([task1, task2])
    result = await manager.execute()
    print(result)

# Run the coroutine
asyncio.run(main())
```

Which of the following statements is true about this code?

A) The tasks are executed concurrently but their results are returned in the order they were added.

B) The tasks are executed sequentially and their results are returned in a list.

C) The tasks are executed concurrently, but there's no guarantee on the order of the results.

D) The tasks will raise an exception because they are not properly defined as coroutines.
    
    ## Answer
    A) The tasks are executed concurrently but their results are returned in the order they were added.

Explanation: The `AsyncManager` class uses asyncio to execute multiple tasks concurrently. Despite being executed concurrently, the `execute` method collects the results in the order they were added to the `tasks` list when creating the instance of `AsyncManager`. This means that the result of `task1()` will be the first element in the returned list, followed by the result of `task2()`, regardless of their execution time.
